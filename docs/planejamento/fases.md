# Fases do Projeto

## Fase 0 - Fundacao

Objetivo: preparar o projeto para desenvolvimento controlado.

Entregas:

- Documentacao inicial.
- Definicao da stack.
- Repositorio Git.
- Estrutura de pastas.
- Padroes de desenvolvimento.
- Decisao sobre servidor local ou nuvem.

## Fase 1 - Nucleo do Sistema

Objetivo: criar a base operacional.

Entregas:

- Login.
- Usuarios.
- Perfis e permissoes.
- Cadastros base.
- Banco de dados inicial.
- Auditoria basica.
- Layout principal do desktop.

## Fase 2 - Comercial e Atendimento

Objetivo: controlar relacionamento e vendas iniciais.

Entregas:

- Clientes.
- Historico de atendimento.
- Orcamentos.
- Pedidos de venda.
- Consulta de produtos.
- Status de negociacao.

## Fase 3 - Estoque e Compras

Objetivo: controlar produtos e reposicao.

Entregas:

- Movimentacao de estoque.
- Entrada e saida.
- Reserva de produtos.
- Inventario.
- Ponto de reposicao.
- Pedido de compra.
- Recebimento.

## Fase 4 - Ordem de Servico

Objetivo: integrar servicos, pecas e clientes.

Entregas:

- Abertura de ordem de servico.
- Pecas utilizadas.
- Servicos executados.
- Tecnicos.
- Status da execucao.
- Encerramento.
- Faturamento a partir da ordem.

## Fase 5 - Financeiro

Objetivo: controlar valores a pagar e receber.

Entregas:

- Contas a receber.
- Contas a pagar.
- Caixa.
- Parcelas.
- Baixas.
- Relatorios financeiros.

## Fase 6 - Relatorios e Gestao

Objetivo: entregar visao gerencial.

Entregas:

- Painel de indicadores.
- Relatorios comerciais.
- Relatorios de estoque.
- Relatorios financeiros.
- Relatorios de ordens de servico.

## Fase 7 - Homologacao e Implantacao

Objetivo: validar o uso real antes da operacao definitiva.

Entregas:

- Testes com usuarios.
- Ajustes de fluxo.
- Treinamento.
- Migracao ou cadastro inicial de dados.
- Backup configurado.
- Plano de suporte.

## Fase 8 - Evolucao Web e Mobile

Objetivo: ampliar canais depois da versao desktop estabilizada.

Entregas:

- Web para consultas e operacoes administrativas.
- Mobile para aprovacoes, consultas e atendimento externo.
- Ajustes de seguranca para acesso remoto.

## Criterio de Avanco Entre Fases

Uma fase so deve ser considerada concluida quando:

- As funcionalidades previstas estiverem implementadas.
- Os testes principais passarem.
- As permissoes estiverem verificadas.
- Os fluxos forem validados com usuario.
- A documentacao essencial estiver atualizada.
