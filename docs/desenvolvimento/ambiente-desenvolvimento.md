# Ambiente de Desenvolvimento

## Sistema Operacional

Desenvolvimento inicial recomendado em Windows, por causa do foco em desktop Windows.

## Ferramentas Recomendadas

- Editor: Cursor ou Visual Studio Code.
- Controle de versao: Git.
- Frontend desktop: Flutter SDK.
- Backend: .NET SDK ou Node.js LTS.
- Banco de dados: PostgreSQL.
- Cliente de banco: DBeaver ou pgAdmin.
- Testes de API: Bruno, Insomnia ou Postman.

## Stack Recomendada

### Opcao Preferencial

- Flutter para desktop, Web e mobile.
- ASP.NET Core para API.
- PostgreSQL para banco de dados.

Motivo: boa performance, tipagem forte, ecossistema maduro e excelente suporte em Windows.

### Opcao Alternativa

- Flutter para desktop, Web e mobile.
- Node.js com NestJS para API.
- PostgreSQL para banco de dados.

Motivo: desenvolvimento rapido e arquitetura modular. Pode ser preferida se a equipe tiver mais experiencia com TypeScript.

## Ambientes

- Desenvolvimento: maquina do desenvolvedor.
- Homologacao: base de testes parecida com a operacao real.
- Producao: servidor local ou remoto usado pelos desktops.

## Configuracao Inicial Recomendada

1. Criar repositorio Git.
2. Definir stack final do backend.
3. Criar projeto desktop.
4. Criar projeto backend.
5. Configurar banco PostgreSQL.
6. Criar migracoes iniciais.
7. Configurar padrao de logs.
8. Configurar testes basicos.
